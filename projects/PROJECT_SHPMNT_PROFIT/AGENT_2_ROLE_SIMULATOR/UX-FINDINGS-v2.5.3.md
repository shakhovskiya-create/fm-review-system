## СЦЕНАРИИ ДНЯ МЕНЕДЖЕРА

### СЦЕНАРИЙ A: Плюсовая сделка (автосогласование)

```
Шаг 1: Открыть ЛС, выбрать позиции — 30 сек
Шаг 2: Система проверяет рентабельность — 5 сек
Шаг 3: Отклонение < 1 п.п. → автосогласование — мгновенно
Шаг 4: Заказ в статусе "Согласован" → WMS — 5 сек
─────────────────────────────────────────────────
ИТОГО: ~1-2 минуты
ОЦЕНКА: Быстро, соответствует таргету 2-5 мин
```

### СЦЕНАРИЙ B: Минусовая сделка (согласование РБЮ)

```
Шаг 1: Открыть ЛС, выбрать позиции — 30 сек
Шаг 2: Система показывает отклонение 5 п.п. — 5 сек
Шаг 3: Отправить на согласование — 5 сек
Шаг 4: ОЖИДАНИЕ РБЮ — до 4 часов
Шаг 5: Получить решение (одобрено/отказ) — ?
─────────────────────────────────────────────────
ИТОГО: от 1 мин до 4 часов
КРИТИЧНО: 4 часа ожидания недопустимо для клиента
```

### СЦЕНАРИЙ C: Срочная сделка

```
Fast-track: "Экстренное согласование"
Ограничения: 3 раза в месяц на менеджера
Условия: письменное подтверждение клиента с датой
Если лимит исчерпан: Только обычный процесс
РИСК: Потеря сделки при исчерпании лимита
```

### СЦЕНАРИЙ D: Конец месяца (пиковая нагрузка)

```
Согласующий завален: 20-30 заявок вместо 5-10
Время ожидания: Растёт (очередь длиннее)
Автоэскалация: Работает, но ДП тоже завален
РИСК: Коллапс согласований в последние дни месяца
```

## ПРОБЛЕМЫ UX

### [UX-001] Нет уведомления о решении согласования

| Параметр | Значение |
|----------|----------|
| **Роль** | Менеджер по продажам |
| **Шаг** | Ожидание согласования |
| **Суть** | Менеджер не получает push/email при согласовании/отказе |
| **Влияние** | Скорость реакции, клиент ждёт, сделка под угрозой |
| **Красный флаг** | Нужно звонить/писать, чтобы узнать статус |

| **Рекомендация** | Push-уведомление + email при изменении статуса |

## КОНФЛИКТЫ РОЛЕЙ

### Менеджер vs РБЮ

| Менеджер хочет | РБЮ хочет |
|----------------|-----------|
| Быстрое согласование (<1ч) | Время на анализ (4ч норма) |
| Понятный отказ | Не объяснять каждый раз |
| Меньше блокировок | Контроль маржи |

**Как решено в ФМ:**
- Автосогласование <1 п.п. — снижает нагрузку на РБЮ
- Автоэскалация при просрочке — мотивирует РБЮ

**Не решено:** Менеджер не видит прогресс согласования

### Менеджер vs Система

| Менеджер хочет | Система требует |
|----------------|-----------------|
| Провести сделку сейчас | Дождаться согласования |
| Обойти правила для KPI | Соблюдать лимиты |
| Меньше ограничений | Защита маржи |

**Риск:** Мотивация к обходу системы
