# Регистр сведений: ekf_ОчередьОтправкиСобытий

**Назначение:** Очередь повторной отправки исходящих событий. При недоступности NiFi события помещаются в этот регистр и обрабатываются фоновым заданием с экспоненциальным backoff.

**Задача:** EKFLAB-518
**Автор:** Шаховский А.С.

## Структура

| Поле | Роль | Тип | Индекс | Описание |
|------|------|-----|--------|----------|
| ИдентификаторСообщения | Измерение | УникальныйИдентификатор | Да | Ключ (UUID v4, = message_id) |
| ТипСобытия | Ресурс | Строка(100) | Нет | `order.created`, `shipment.posted` и т.д. |
| Топик | Ресурс | Строка(100) | Нет | Kafka-топик (`1c.order.created.v1`) |
| ТелоСообщения | Ресурс | ХранилищеЗначения | Нет | JSON payload (сериализованный в строку) |
| ИдКорреляции | Ресурс | Строка(36) | Нет | UUID корреляции (X-Correlation-Id) |
| ДатаСоздания | Ресурс | Дата | Нет | Время постановки в очередь |
| КоличествоПопыток | Ресурс | Число(3,0) | Нет | Счетчик попыток (макс. 100) |
| СледующаяПопытка | Ресурс | Дата | Нет | Дата/время следующей попытки (backoff) |
| КодОшибки | Ресурс | Число(5,0) | Нет | Последний HTTP-код (0 = сетевая ошибка) |
| ТекстОшибки | Ресурс | Строка(500) | Нет | Описание последней ошибки |
| Статус | Ресурс | Строка(20) | Нет | `pending` / `sending` / `expired` |

## Жизненный цикл записи

```
1. Событие не отправлено -> ПоместитьВОчередь() -> Статус = "pending"
2. Фоновое задание берет -> ОбновитьСтатусЗаписи("sending")
3a. Отправка успешна -> УдалитьЗаписьИзОчереди()
3b. Отправка неуспешна:
    - Попытки < МаксПопыток -> Статус = "pending", СледующаяПопытка = backoff
    - Попытки >= МаксПопыток -> Статус = "expired" (алерт в журнал)
4. Очистка: expired записи старше 7 дней удаляются
```

## Exponential backoff

| Попытка | Задержка |
|---------|----------|
| 1 | 30 сек |
| 2 | 1 мин |
| 3 | 2 мин |
| 4 | 5 мин |
| 5 | 10 мин |
| 6-10 | 15 мин |
| 11-20 | 30 мин |
| 21-50 | 1 час |
| 51-100 | 2 часа |
| >100 | expired |
