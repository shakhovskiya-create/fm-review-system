**Confluence:** page 83951683, version 44 -> 45 (v1.0.5)

## Решения пользователя

| Вопрос | Решение |
|--------|---------|
| Объем правок | Все P1+P2+P3 (25 рекомендаций) |
| Система маршрутизации | ELMA (по протоколу 26.12) вместо 1С:ДО |

## Примененные изменения

### P1 - Блокеры

| ID | Где | Было | Стало | Причина |
|----|-----|------|-------|---------|
| P1-01 | Новый раздел 3.19 | Отсутствовал | Раздел "Подготовка и актуализация НПСС": методология, ответственные, сроки, критерии готовности, состав, актуализация, контроль качества | НПСС - фундамент всех формул. Без описания процесса заполнения система не может работать |
| P1-02 | п. 3.4-3.8, 3.11, 3.12, 3.7 | "1С:ДО" (12 упоминаний) | "ELMA" | Протокол 26.12 зафиксировал переход на ELMA |
| P1-03a | п. 3.3 (таблица показателей) | 5 показателей | 6-й: "Мин. допустимая цена (руб.)" с формулой | Менеджер мыслит ценами, не % рентабельности |
| P1-03b | п. 3.2 (принципы) | Сообщение только в п.п. | Дублирование в рублях: "цена ниже минимальной на 850 руб." | UX: менеджеру понятнее рубли |
| P1-04 | п. 3.1 (концепция) | "floor price - отдельный процесс, вне scope" | Рекомендательный (не блокирующий) контроль: предупреждение при рент. < 5%, уведомление РБЮ | Без контроля на входе система ловит проблемы слишком поздно |
| P1-05a | п. 3.6 | Единый SLA для всех | Дифференциация: P1 (критические > 500 т.р.) = 4ч / P2 (стандартные) = 24ч | Противоречие с протоколом 26.12 (48ч). Компромисс |
| P1-05b | п. 3.4 | "РБЮ не ответил за 4 часа" | "4 часа (P1) или 24 часа (P2)" | Согласованность с п. 3.6 |

### P2 - Существенные

| ID | Где | Было | Стало | Причина |
|----|-----|------|-------|---------|
| P2-01 | п. 3.4 (BPMN + таблица) | Два варианта: Согласовать / Отклонить | Три варианта: + "Согласовать с корректировкой цены" (желтый статус) | Требование протокола 26.12. Убирает лишний цикл отказ→новый Заказ |
| P2-02 | п. 3.14 | Нет защиты от выборочных возвратов | Правило: при падении рент. пакета ниже согласованного порога - уведомление РБЮ. При возврате > 30% стоимости - обязательный пересчет | Уязвимость: возврат высокомаржинальных позиций |

| P2-09 | п. 3.12 | Нет автозакрытия просроченных ЛС | Автоматический перевод в "Закрыта" через 5 дней после истечения, уведомление за 3 дня | "Мертвые" ЛС накапливаются |
