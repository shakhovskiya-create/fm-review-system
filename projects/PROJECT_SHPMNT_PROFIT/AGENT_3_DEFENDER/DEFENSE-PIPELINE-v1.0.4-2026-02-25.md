### #17: Q3 / GAP-02 — Race condition: одновременный пересчет НПСС и согласование ЛС

**Источник:** Agent 4 (QA), Agent 1 (GAP-02)
**Серьезность:** LOW (GAP)
**Тип:** A (Учтено)
**Статус:** Покрыто существующими механизмами

**На самом деле:** ФМ v1.0.4 уже содержит механизмы, делающие race condition безопасной:

1. **LS-BR-035 (блокировка):** При создании/редактировании Заказа ЛС блокируется пессимистичной блокировкой (5 мин). Пересчет НПСС не может изменить данные заблокированной ЛС.

2. **НПСС зафиксирована при создании ЛС** (п. 3.3): «НПСС -- зафиксированная стоимость на дату создания ЛС». Пересчет НПСС по автотриггерам (LS-BR-075) создает **новую** фиксацию с пересмотром (5 р.д.), а не изменяет текущую запись.

3. **Атомарная проверка рентабельности** (п. 3.6): Расчет рентабельности Заказа выполняется атомарно при проведении, используя зафиксированную НПСС.

**Вывод:** Согласование ЛС и пересчет НПСС работают с разными версиями данных. Пересчет НПСС по триггерам создает новую фиксацию (со сроком пересмотра 5 р.д.), а не модифицирует текущую. Race condition невозможна при текущей архитектуре. Дополнительных правок не требуется.
