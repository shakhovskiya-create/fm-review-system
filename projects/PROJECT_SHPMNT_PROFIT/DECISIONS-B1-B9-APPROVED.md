# Утвержденные решения по бизнес-вопросам B1-B9
## FM-LS-PROFIT v1.0.6: Подготовка к обновлению v1.0.7

**Дата утверждения:** 13.02.2026
**Проект:** Контроль рентабельности при частичных отгрузках по ЛС
**Статус:** ✅ УТВЕРЖДЕНО стейкхолдерами
**Влияние на ТЗ:** +68 часов (+3% от базовой оценки 2293ч)

---

## Резюме

Все 9 бизнес-вопросов, выявленных в ходе полного цикла конвейера (Agents 1-6), рассмотрены и утверждены стейкхолдерами.

**Результат:** Готовность к обновлению ФМ v1.0.6 → v1.0.7 с внесением уточнений по всем 9 вопросам.

---

## Утвержденные решения

### B1 (CRITICAL): SLA для заказов < 100 тыс. руб.

**Проблема:** Противоречие между "P2 вдвое" (12ч/24ч/36ч) и фактическими значениями (2ч/4ч/12ч).

**✅ УТВЕРЖДЕНО:** Жесткие значения 2ч/4ч/12ч

**Решение:**
- РБЮ: 2 часа
- Директор ДРП: 4 часа
- ДП: 12 часов

**Дельта трудоемкости:** +8 часов

**Обоснование:** Минимальная дельта, точное соответствие ФМ, сохранение связи с приоритетом P2.

---

### B2 (CRITICAL): Согласование с корректировкой цены

**Проблема:** Нет описания полного цикла: автозамена, рекурсия, EDI-ограничения.

**✅ УТВЕРЖДЕНО:** Рекомендация менеджеру

**Решение:**
1. Согласующий указывает рекомендуемую цену
2. Менеджер вносит вручную
3. Система автоматически перепроверяет
4. Автосогласование если улучшение достаточное

**Дельта трудоемкости:** +8 часов

**Обоснование:** Баланс автоматизации и контроля, менеджер остается в контуре решения.

---

### B3 (CRITICAL): Логический резерв ЛС при дефиците

**Проблема:** Противоречие 48ч vs 90 дней, блокирует vs не блокирует.

**✅ УТВЕРЖДЕНО:** 48ч блокирует + 90д информационно

**Решение:**
- **Блокирующий резерв:** 48 часов для срочных заказов
- **Информационная фиксация:** до 90 дней для планирования закупок

**Дельта трудоемкости:** 0 часов

**Обоснование:** Баланс скорости продаж и контроля дефицита, не тормозит основной процесс.

---

### B4 (CRITICAL): Фоновое задание пересчета санкций

**Проблема:** Нет retry/timeout/мониторинга.

**✅ УТВЕРЖДЕНО:** Полная спецификация

**Решение:**
- **Retry:** 3 попытки каждые 15 минут
- **Таймаут:** 2 часа
- **Мониторинг:** Уведомление финансовому директору при сбое

**Дельта трудоемкости:** +8 часов

**Обоснование:** Санкции - критичная бизнес-логика (деньги компании), сбой должен быть виден ФД немедленно.

---

### B5 (HIGH): Терминальное поведение при автоотказе ГД

**Проблема:** После автоотказа ГД (48ч) нет эскалации выше - "черная дыра" для крупных сделок.

**✅ УТВЕРЖДЕНО:** Уведомление CFO + пометка

**Решение:**
- Уведомление финансовому директору (CFO)
- Пометка "Автоматический отказ (превышение SLA)"
- Возможность повторной подачи менеджером
- Возможность ручного одобрения CFO через арбитраж

**Дельта трудоемкости:** +12 часов

**Обоснование:** Крупные сделки (требующие ГД) не должны "зависать", CFO должен иметь видимость.

---

### B6 (HIGH): Улучшение рентабельности во время согласования

**Проблема:** Если отклонение уменьшилось (с -5% до -0.5%) - нет правила снижения уровня.

**✅ УТВЕРЖДЕНО:** Автосогласование при отклонении < 1 п.п.

**Решение:**
- Если отклонение рентабельности уменьшается до < 1 п.п. → автосогласование
- Уведомление согласующему о факте автосогласования
- Фиксация в журнале аудита

**Дельта трудоемкости:** +8 часов

**Обоснование:** Поощрение улучшения рентабельности, ускорение процесса согласования.

---

### B7 (HIGH): Формулы "Выручка отгруженного" - нетто или брутто

**Проблема:** ФМ не уточняет, учитываются ли возвраты в формуле.

**✅ УТВЕРЖДЕНО:** Нетто (с учетом возвратов за 60 дней)

**Решение:**
- **Выручка отгруженного (нетто)** = Отгружено - Возвращено (за 60 дней)
- Автоматический пересчет при каждом возврате

**Дельта трудоемкости:** +8 часов

**Обоснование:** Реальная рентабельность должна учитывать возвраты для корректного отслеживания cherry-picking.

---

### B8 (HIGH): Автопродление ЛС + НПСС > 90 дней

**Проблема:** Автопродление при позднем поступлении может конфликтовать с блокировкой по НПСС.

**✅ УТВЕРЖДЕНО:** Блокировка остается независимой

**Решение:**
- Автопродление ЛС срабатывает
- Блокировка по НПСС > 90 дней действует независимо
- Менеджер видит предупреждение о блокировке

**Дельта трудоемкости:** 0 часов

**Обоснование:** НПСС > 90 дней - критичный риск неликвидов, автопродление не должно обходить контроль.

---

### B9 (HIGH): Определение подтвержденного дефицита

**Проблема:** ФМ описывает ручное определение РБЮ, но правила алгоритмические - неоднозначность.

**✅ УТВЕРЖДЕНО:** Автоматическое определение + РБЮ только для споров

**Решение:**
- Система автоматически определяет дефицит по правилам
- РБЮ вмешивается только при споре (SLA 3 рабочих дня)
- Менеджер может оспорить решение системы

**Дельта трудоемкости:** +16 часов

**Обоснование:** Снижение нагрузки на РБЮ, ускорение процесса, споры - редкость.
