## О чём эта папка

Здесь собраны все материалы по оптимизации процесса **проектных продаж**:
- Диагностика проблем
- Мини-ФМ (функциональные модели) по каждому решению
- Технические задания для разработки

**Контекст:** Проектные продажи составляют ~50% бизнеса. План следующего года на 70% зависит от эффективности управления этим процессом.

## Выявленные проблемы

| Код | Проблема | Симптом |
|-----|----------|---------|
| PIPE-A | Низкая конверсия воронки | 100 млрд пайплайн → 50 млн отгрузки (0.05%) |
| PIPE-B | Низкий % выкупа | 40-46% в месяц |
| PIPE-C | Нет пайплайна на год | Запас 7% при конверсии 22% |
| PIPE-D | Смешение типов заявок | Подбор аналогов = Проект |
| PIPE-E | Неактуальность данных | ТКП не знает о проекте |
| PIPE-F | Множественные сметы | Нельзя считать рентабельность |

## Порядок внедрения

### Итерация 1 (Quick Wins)
- FM-003: Подбор аналогов — быстрый эффект, ТЗ уже готово
- FM-001: Воронки — немедленная видимость горячих проектов

### Итерация 2 (Foundation)
- FM-002: Стадии — даёт взвешенный forecast
- FM-004: Автозакрытие — автоматизирует актуальность

### Итерация 3 (Complex)
- FM-005: Мастер-смета — требует миграции данных
