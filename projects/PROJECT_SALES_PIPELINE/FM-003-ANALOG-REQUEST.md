## 2. ПРОБЛЕМА (AS-IS)

### 2.1 Текущее состояние

```
В 1С НЕ разделены:
- Простой подбор аналогов (сервис "за 4 часа")
- Полноценные проекты

Инженеры ВЫНУЖДЕНЫ создавать проект для ЛЮБОГО подбора.
```

### 2.2 Последствия

| Проблема | Влияние |
|----------|---------|
| Засорение воронки | Проекты-однодневки портят статистику |
| Искажение конверсии | Подборы закрываются быстро → завышенная конверсия |
| Нагрузка на инженеров | Заполнение полной карточки ради 1 вопроса |
| Потеря фокуса | Менеджеры работают с мусором вместо реальных проектов |

## 4. БИЗНЕС-ПРАВИЛА

### BR-001: Создание заявки
```
КОГДА: Создание новой заявки на подбор
ОБЯЗАТЕЛЬНЫЕ ПОЛЯ:
  - Клиент (контрагент)
  - Контактное лицо
  - Что ищем (текстовое описание)
  - Срочность (Обычная / Срочная)

АВТОЗАПОЛНЕНИЕ:
  - Менеджер = текущий пользователь
  - Статус = "Новая"
  - Дата создания = сейчас
```

### BR-002: Назначение инженера
```
КОГДА: Статус меняется на "В работе"
ТРЕБУЕТСЯ: Указать инженера

ОПЦИОНАЛЬНО: Автораспределение по очереди
  - Найти инженера с минимальной нагрузкой
  - Назначить автоматически
  - Отправить уведомление
```

### BR-003: Выполнение заявки
```
КОГДА: Инженер завершил подбор
ДЕЙСТВИЯ:
  - Заполнить поле "Результат"
  - Сменить статус на "Выполнена"
  - Уведомить менеджера
```

### BR-004: Перевод в проект
```
КОГДА: Подбор перерос в полноценный проект
ДЕЙСТВИЯ:
  1. Нажать кнопку "Создать проект"
  2. Система создаёт проект с данными:
     - Клиент ← из заявки
     - Контакт ← из заявки
     - Описание потребности ← "Что ищем"
     - Источник = ссылка на заявку
     - Стадия = "Лид" (или "Квалификация")
  3. Заявка получает статус "Переведена в проект"
  4. В заявке — ссылка на созданный проект
```

### BR-005: Автозакрытие
```
РЕГЛАМЕНТНОЕ ЗАДАНИЕ: Ежедневно в 00:00

НАЙТИ заявки где:
  - Статус = "Новая" ИЛИ "В работе"
  - (Сегодня - Дата_последней_активности) ≥ 7 дней

ДЕЙСТВИЯ:
  - Статус = "Закрыта автоматически"
  - Записать причину: "Автозакрытие: нет активности 7 дней"
  - Уведомить менеджера
```

### BR-006: Что считается активностью
```
Активность — это:
  - Изменение любого поля заявки
  - Добавление комментария
  - Смена статуса

НЕ активность:
  - Просмотр заявки
```

## 5. ИЗМЕНЕНИЯ В 1С

### 5.1 Документ "ЗаявкаНаПодборАналогов"

| Реквизит | Тип | Обязательность | Описание |
|----------|-----|----------------|----------|
| Номер | Строка(11) | Авто | ПА-NNNNNN |
| Дата | Дата | Авто | Дата создания |
| Клиент | Справочник.Контрагенты | Да | Кто запросил |
| КонтактноеЛицо | Справочник.КонтактныеЛица | Да | С кем общаться |
| ЧтоИщем | Строка(1000) | Да | Описание запроса |
| Срочность | Перечисление.Срочность | Да | Обычная / Срочная |
| Менеджер | Справочник.Пользователи | Авто | Кто создал |
| Инженер | Справочник.Пользователи | При "В работе" | Кто выполняет |
| Статус | Перечисление.СтатусыЗаявки | Да | Текущий статус |
| Результат | Строка(2000) | При "Выполнена" | Ответ инженера |
| СвязанныйПроект | Справочник.Проекты | — | Если переросла |
| ДатаПоследнейАктивности | Дата | Авто | Для автозакрытия |
| ПричинаЗакрытия | Строка(200) | При закрытии | Почему закрыта |

### 5.2 Перечисление "СтатусыЗаявкиПодбора"

| Значение | Представление | Переходы из |
|----------|---------------|-------------|
| Новая | Новая | — |
| ВРаботе | В работе | Новая |
| Выполнена | Выполнена | В работе |
| Отменена | Отменена | Новая, В работе |
| ПереведенаВПроект | Переведена в проект | В работе, Выполнена |
| ЗакрытаАвтоматически | Закрыта автоматически | Новая, В работе |

### 5.3 Перечисление "Срочность"

| Значение | Представление | SLA |
|----------|---------------|-----|
| Обычная | Обычная | 4 часа |
| Срочная | Срочная (2ч) | 2 часа |

### 5.4 Регламентное задание "АвтозакрытиеЗаявокПодбора"

