# Структура шаблона FM-страницы в Notion

## Как это работает

```
┌─────────────────────────────────────────────────────────────┐
│  ШАБЛОН FM-СТРАНИЦЫ                                         │
│                                                             │
│  Создается ОДИН РАЗ вручную в Notion с:                    │
│  - Linked Database views (Глоссарий, Требования, Риски)    │
│  - Фильтры: ФМ contains {{FM_CODE}}                        │
│  - Placeholder'ы для контента                               │
│                                                             │
│  При миграции скрипт:                                       │
│  1. Дублирует шаблон                                        │
│  2. Заполняет БД записями со связью на новую страницу      │
│  3. Linked views автоматически показывают данные            │
└─────────────────────────────────────────────────────────────┘
```

## Структура шаблона

```
📄 FM-TEMPLATE (страница-шаблон)
│
├── 📋 ПАСПОРТ ДОКУМЕНТА
│   └── [таблица: Версия, Дата, Статус]
│
├── 📖 ГЛОССАРИЙ
│   └── [LINKED DATABASE: БД Глоссарий, фильтр ФМ = текущая страница]
│       View: Table, колонки: Термин, Определение, Категория
│
├── 📝 СОДЕРЖАНИЕ ДОКУМЕНТА
│   └── [текст, заголовки, списки, таблицы - из Word]
│
├── 📋 РЕЕСТР ТРЕБОВАНИЙ
│   └── [LINKED DATABASE: БД Требования, фильтр ФМ = текущая страница]
│       View: Table, колонки: Код, Название, Тип, Статус, Приоритет
│       Группировка: по Типу (BR, FR, WF, RPT, NFR, INT, SEC)
│
├── ⚠️ РИСКИ
│   └── [LINKED DATABASE: БД Риски, фильтр ФМ = текущая страница]
│       View: Table, колонки: Название, Вероятность, Влияние, Статус
│
└── 📜 ИСТОРИЯ ВЕРСИЙ
    └── [LINKED DATABASE: БД История версий, фильтр ФМ = текущая страница]
        View: Table, колонки: Версия, Дата, Изменения
```

## Преимущества

1. **Linked Views работают сразу** - не надо настраивать вручную каждый раз
2. **Фильтры автоматические** - показываются только записи этой ФМ
3. **Единый формат** - все ФМ выглядят одинаково
4. **Можно редактировать требования** прямо на странице ФМ

## Как создать шаблон

1. Создать новую страницу в Notion
2. Добавить структуру (заголовки разделов)
3. В каждом разделе добавить `/linked` → выбрать БД
4. Настроить view (колонки, сортировка)
5. Добавить фильтр: ФМ contains "FM-" (будет работать для всех ФМ)
6. Сохранить как шаблон БД "Функциональные модели"
