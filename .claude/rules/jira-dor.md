# Definition of Ready (DoR) — входная проверка качества задачи

## Обязательные поля при создании задачи

`jira-tasks.sh create` проверяет DoR и выдает WARNING если поля не заполнены:

| Поле | Флаг | Пример |
|------|------|--------|
| Sprint | `--sprint N` | `--sprint 28` |
| Проблема | в `--body` | "h2. Проблема" |
| Решение | в `--body` | "h2. Решение" |
| Component | `--component X` | `--component profitability-service` |
| Fix Version | `--version X` | `--version v0.4.0` |

## DoR != блокер

DoR выдает WARNING, но не блокирует создание задачи. Это напоминание, а не стоп.
Причина: иногда нужно быстро создать задачу-заглушку с минимумом данных.

## Полный пример создания задачи

```bash
jira-tasks.sh create \
  --title "Интегрировать Claude API для интеллектуальной аналитики" \
  --agent 12-dev-go \
  --sprint 29 \
  --component analytics-service \
  --version v0.5.0 \
  --priority medium \
  --parent EKFLAB-6 \
  --body "h2. Проблема
Сейчас аналитика рентабельности работает только на уровне формул и пороговых правил. Нет интеллектуального анализа аномалий — менеджер вынужден сам разбираться в причинах отклонений.

h2. Решение
Интегрировать Claude API (anthropic-sdk-go) для двух уровней аналитики:
* Sonnet 4.6 — объяснение аномалий на человеческом языке (90% запросов)
* Opus 4.6 — автономное расследование причин отклонений (10% запросов)
* Кэширование системного промпта (экономия 90% на повторных запросах)
* Учёт стоимости каждого запроса (лимит 50 USD/день)

h2. Ожидаемый результат
Go-клиент для Claude API в internal/adapter/claude/ — готов к использованию analytics-service.

h2. Эффект
ФД и менеджеры получают объяснения аномалий без ручного анализа. Время разбора инцидента: часы -> минуты.

h2. Scope
Файлы: profitability-service/internal/adapter/claude/"
```

**AC → в Smart Checklist** (не в описании):
```
---
- AC: Интеграция Sonnet для Level 2 аналитики
- AC: Интеграция Opus для Level 3 расследований
- AC: Кэширование системного промпта реализовано
- AC: Structured output парсинг работает
- AC: Учёт стоимости каждого запроса
---
- Тесты проходят
- Регрессий нет
- Критерии приёмки выполнены
- Артефакты перечислены
- Документация обновлена (или N/A)
- Скрытого техдолга нет
```
