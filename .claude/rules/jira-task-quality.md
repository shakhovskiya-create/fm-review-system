# Jira Task Quality Standard

## Description Format (ОБЯЗАТЕЛЬНО)

Jira Server НЕ поддерживает Markdown. Описания ТОЛЬКО в Jira Wiki Markup.

### Конвертация MD → Wiki

| Markdown | Jira Wiki |
|----------|-----------|
| `## Заголовок` | `h2. Заголовок` |
| `### Подзаголовок` | `h3. Подзаголовок` |
| `- Пункт` | `* Пункт` |
| `` `код` `` | `{{код}}` |
| `**жирный**` | `*жирный*` |

### Шаблон описания (wiki markup)

```
h2. Проблема
[Какую проблему решаем — 1-3 предложения, понятным бизнес-языком на русском]

h2. Решение
[Что конкретно делаем — список шагов/компонентов]

h2. Ожидаемый результат
[Что появится/изменится после завершения — конкретный артефакт]

h2. Эффект
[Как это влияет на бизнес/пользователей — зачем это нужно]

h2. Scope
Файлы: [список затрагиваемых файлов/директорий]
```

### Правила описания
- **Язык:** Русский. Заголовки, проблема, решение, результат, эффект — всё на русском
- **AC → Smart Checklist:** Критерии приёмки НЕ в описании, а в Smart Checklist (плагин)
- **Нет чекбоксов в описании:** `* ( )` / `* (x)` — запрещены. Чекбоксы = Smart Checklist
- **Нет ссылок на GitHub Issues:** `#123`, `Migrated from` — удалять при обнаружении
- **Проблема обязательна:** Без "Проблема" непонятно зачем задача существует
- **Эффект обязателен:** Привязка к бизнес-ценности ("ускоряет/упрощает/экономит")

### Запрещено в описании
- `## ` (markdown заголовок) — использовать `h2.`
- `- [ ]` / `* ( )` (чекбоксы) — AC только в Smart Checklist
- `**text**` (markdown bold) — использовать `*text*`
- Английский язык в заголовках и описании
- Ссылки на GitHub Issues (`#NNN`, `Migrated from`)
- Пустые секции ("TBD", "TODO")

## Smart Checklist (ОБЯЗАТЕЛЬНО)

Smart Checklist содержит **две секции**: критерии приёмки (AC) + DoD.

### При создании задачи

`jira-tasks.sh create` автоматически ставит стандартный DoD. **Агент добавляет AC конкретной задачи:**

```
---
- AC: Интеграция Sonnet для Level 2 аналитики
- AC: Интеграция Opus для Level 3 расследований
- AC: Кэширование системного промпта реализовано
- AC: Учёт стоимости каждого запроса
---
- Тесты проходят
- Регрессий нет
- Критерии приёмки выполнены
- Артефакты перечислены
- Документация обновлена (или N/A)
- Скрытого техдолга нет
```

**Правила AC:**
- На русском языке
- Конкретные, проверяемые (не "всё работает", а "Sonnet отвечает за <15 секунд")
- 3-7 пунктов (не больше — иначе задачу надо декомпозировать)
- Разделитель `---` между AC и DoD

### При закрытии задачи
Все пункты ДОЛЖНЫ быть:
- `+` (checked/DONE) — пункт выполнен
- `x` (skipped) — пункт неприменим (например "Документация обновлена" → `x Документация обновлена: N/A`)

ЗАПРЕЩЕНО оставлять `-` (unchecked) при закрытии. `jira-tasks.sh done` блокирует.

### Кастомизация Smart Checklist
Если задача требует специфических проверок DoD — ДОБАВИТЬ пункты (не удалять стандартные):
```
---
- AC: ...
---
- Тесты проходят
- Регрессий нет
- Критерии приёмки выполнены
- Артефакты перечислены
- Документация обновлена (или N/A)
- Скрытого техдолга нет
- SE ревью пройдено (специфика)
- Confluence опубликован (специфика)
```

## Валидация (автоматическая)

`jira-tasks.sh done` при закрытии проверяет:
1. `--comment` обязателен (exit 1 без него)
2. Smart Checklist: все `-` → ошибка (exit 1)
3. Artifact cross-check: файлы из `git diff` vs текст комментария

## Процесс разработки (end-to-end)

### Создание задачи
1. `jira-tasks.sh create --title "..." --body "..."` — описание НА РУССКОМ в wiki markup
2. Smart Checklist ставится автоматически (6 пунктов, все `-`)
3. Если нужны доп. пункты — добавить через API сразу

### Работа над задачей
1. `jira-tasks.sh start EKFLAB-N` → статус "В работе"
2. По мере работы: обновлять Smart Checklist пункты на `+`
3. Если пункт N/A: обновить на `x` с пояснением

### Закрытие задачи
1. Убедиться: все пункты `+` или `x` (нет `-`)
2. `jira-tasks.sh done EKFLAB-N --comment "Результат: ... Файлы: ..."`
3. Комментарий: РЕЗУЛЬТАТ + Было→Стало (НЕ дублировать DoD, он в чеклисте)

### Epic
1. Подзадачи закрываются по одной с полным DoD
2. Epic закрывается ПОСЛЕДНИМ — скрипт проверяет children
3. Epic Smart Checklist: `+ All children closed`, `+ AC met`, etc.
