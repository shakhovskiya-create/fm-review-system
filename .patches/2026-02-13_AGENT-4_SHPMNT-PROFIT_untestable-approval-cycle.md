# Непротестируемый цикл согласования из-за неописанных ветвей решения

**Дата:** 2026-02-13
**Агент:** Agent 4 (QA Tester)
**Проект:** PROJECT_SHPMNT_PROFIT
**Версия ФМ:** 1.0.6
**Severity:** CRITICAL

## Паттерн ошибки
ФМ описывает вариант решения согласующего (например, "Согласовать с корректировкой"), но не описывает полный цикл после выбора этого варианта. QA не может сгенерировать тест-кейс, потому что ожидаемый результат не определен.

## Конкретный пример
"Согласовать с корректировкой цены" (п. 3.4) - нет описания: кто вводит новую цену, пересчитывается ли рентабельность, нужна ли повторная проверка, что если новая цена все равно не проходит порог. 3 UX-находки подтвердили критичность (UX-ROLE-013, UX-APPROVER-001, аудит B2).

## Как должно быть
Для каждого варианта решения согласующего - полное описание дальнейшего цикла с условиями выхода. Каждая ветвь должна быть протестируемой (четкий ожидаемый результат).

## Правило для предотвращения
ЕСЛИ в ФМ описан вариант решения в бизнес-процессе (выбор из нескольких действий), ПРОВЕРИТЬ что для КАЖДОГО варианта описан полный цикл до терминального состояния. Отсутствие описания хотя бы одной ветви = CRITICAL.
