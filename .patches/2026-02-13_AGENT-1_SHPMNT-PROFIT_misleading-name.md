# Название показателя/поля не соответствует его фактическому содержанию

**Дата:** 2026-02-13
**Агент:** Agent 1 (Architect)
**Проект:** PROJECT_SHPMNT_PROFIT
**Версия ФМ:** 1.0.1
**Severity:** CRITICAL

## Паттерн ошибки
Название расчетного показателя или поля включает компоненты, которые фактически НЕ входят в его расчет. Разработчик реализует по названию (включая все упомянутые компоненты), а не по формуле/описанию.

## Конкретный пример
Показатель "Накопл. + Заказы На согласовании + Заказ" - название подразумевает включение заказов "На согласовании", но NOTE явно исключает их из расчета. Разработчик включит их в формулу, ориентируясь на название.

## Как должно быть
Название показателя точно отражает его состав. Если показатель НЕ включает компонент - этот компонент НЕ упоминается в названии. Использовать техническое имя + описание состава отдельно.

## Правило для предотвращения
ЕСЛИ название показателя/поля перечисляет компоненты (через "+" или запятую), ПРОВЕРИТЬ что КАЖДЫЙ компонент из названия действительно участвует в расчете. Несоответствие = CRITICAL.
