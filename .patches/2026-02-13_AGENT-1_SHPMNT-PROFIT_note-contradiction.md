# Противоречие числовых значений между основным текстом и NOTE/примечанием

**Дата:** 2026-02-13
**Агент:** Agent 1 (Architect)
**Проект:** PROJECT_SHPMNT_PROFIT
**Версия ФМ:** 1.0.1
**Severity:** CRITICAL

## Паттерн ошибки
В одном разделе ФМ основной текст содержит одно числовое значение параметра (срок, процент, порог), а NOTE/примечание к этому же разделу - другое. Разработчик не может определить правильное значение и реализует произвольный вариант.

## Конкретный пример
Раздел 3.18: основной текст и матрица говорят "6 мес. без нарушений - санкции снимаются". NOTE того же раздела: "Полное снятие через 3 месяца (квартал)". Два взаимоисключающих значения для одного параметра.

## Как должно быть
Единственное значение параметра в основном тексте. NOTE используется только для пояснения (как считать, кто отвечает), но НЕ для переопределения значений из основного текста.

## Правило для предотвращения
ЕСЛИ в NOTE/примечании упоминается числовое значение (срок, процент, сумма, порог), ПРОВЕРИТЬ что оно совпадает с основным текстом раздела. Любое расхождение = CRITICAL.
