# Отсутствие терминального состояния в цепочке эскалации

**Дата:** 2026-02-13
**Агент:** Agent 5 (Tech Architect)
**Проект:** PROJECT_SHPMNT_PROFIT
**Версия ФМ:** 1.0.6
**Severity:** CRITICAL

## Паттерн ошибки
Цепочка эскалации заканчивается на последнем уровне (ГД), но не описано, что происходит при его бездействии. Автоотказ создает "черную дыру" - нет эскалации выше, нет уведомления руководства, процесс зависает.

## Конкретный пример
Автоотказ ГД через 48 часов (п. 3.6): Заказ отклонен, но менеджер не знает, что это автоматический отказ. Нет уведомления CFO/совета директоров. Нет механизма пересмотра. Крупные сделки теряются.

## Как должно быть
Каждая цепочка эскалации должна иметь терминальное состояние с определенными последствиями. Автоотказ верхнего уровня = условный отказ + уведомление вышестоящей инстанции + возможность пересмотра.

## Правило для предотвращения
ЕСЛИ описана цепочка эскалации с таймаутами, ПРОВЕРИТЬ наличие терминального состояния для последнего уровня. Автоотказ без определенных последствий = CRITICAL.
