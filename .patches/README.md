# Шаблон патча (Self-Improvement)

> Этот файл - справка. Реальные патчи создаются агентами автоматически.

## Формат имени файла
```
YYYY-MM-DD_AGENT-N_PROJECT-NAME_CATEGORY.md
```
Пример: `2026-02-13_AGENT-1_SHPMNT-PROFIT_formula-contradiction.md`

## Структура патча

```markdown
# [Краткое описание паттерна ошибки]

**Дата:** YYYY-MM-DD
**Агент:** Agent N (Name)
**Проект:** PROJECT_NAME
**Версия ФМ:** X.Y.Z
**Severity:** CRITICAL / HIGH / MEDIUM

## Паттерн ошибки
Что именно было не так. Обобщенно, без привязки к конкретному разделу.

## Конкретный пример
Где нашли: раздел, пункт. Что было написано. Почему это проблема.

## Как должно быть
Правильная формулировка / структура / подход.

## Правило для предотвращения
Четкое правило, которое агент должен проверять в будущем.
Формат: "ВСЕГДА проверяй..." / "ЕСЛИ ... ТО это ошибка"

## Теги
`#категория` `#подкатегория`
```

## Категории тегов

| Тег | Описание |
|-----|----------|
| `#formula` | Противоречия в формулах, расчетах |
| `#cross-ref` | Битые/неконсистентные ссылки между разделами |
| `#boundary` | Не описаны граничные условия |
| `#terminology` | Неконсистентные термины, определения |
| `#missing-field` | Пропущены обязательные поля/разделы |
| `#ux-flow` | Проблемы пользовательского сценария |
| `#role-access` | Ошибки в ролевой модели / правах |
| `#1c-specific` | Специфика платформы 1С не учтена |
| `#integration` | Проблемы интеграции между подсистемами |
| `#approval-flow` | Ошибки в цепочке согласования |
