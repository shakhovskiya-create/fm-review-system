# Продуктовые команды ДИТ

> Источник: Confluence EW > ДИТ > Продуктовые команды

## Структура команд

| Команда | Зона ответственности | Связь с нашим проектом |
|---------|---------------------|------------------------|
| **Product Team 1C** | Все базы 1С (УТ, ERP, БП, ЗУП, ДО, УХ) | **Прямая** — разработка расширения в 1С:УТ 10.2 |
| **Product Team Sales Web** | Сайт, веб-продажи, API 2.0, EKF Market | Косвенная — EDI-заказы, API для цен/остатков |
| **Product Team FD OD MD** | Финансовый, операционный, управляющий директора | Косвенная — отчётность, дашборды руководства |
| **Product Team BP** | Бизнес-процессы (ELMA, BPM) | **Прямая** — ELMA для согласования отгрузок |
| **Product Team Architect** | Архитектура ИТ-ландшафта, шина данных | Косвенная — целевая архитектура, Kafka |
| **IT Support** | Поддержка пользователей (1/2 линия) | Косвенная — обучение, поддержка пользователей при внедрении |
| **Data Analysis** | Аналитика данных | Косвенная — отчёты по рентабельности |

## Product Team 1C — наша основная команда

Отвечает за все доработки во всех базах 1С.

**Ключевые проекты команды:**
- Себестоимость (SBS) — 9 ЧТЗ
- Контроль рентабельности (наш проект) — FM-LS-PROFIT
- Честный знак — маркировка товаров
- QC (контроль качества) — EproQc

**Стек:**
- 1С:Предприятие 8.3 (платформа)
- 1С:УТ 10.2 (конфигурация, обычные формы)
- Расширения (.cfe)
- HTTP-сервисы 1С

## Product Team BP — согласование через ELMA

Отвечает за BPM-процессы в ELMA.

**Для нашей ФМ:**
- Маршруты согласования отгрузок (РБЮ, ФД, ГД)
- Резервный режим ELMA при недоступности
- Интеграция 1С:УТ ↔ ELMA (API)

## Зависимости между командами (для нашей ФМ)

```
Product Team 1C (расширение 1С:УТ)
        │
        ├── Product Team BP (ELMA согласование)
        │
        ├── Product Team Sales Web (EDI, API)
        │
        └── Product Team FD OD MD (отчёты ФД)
```

Координация через Jira (проекты SBS, SHPMNT-PROFIT).
